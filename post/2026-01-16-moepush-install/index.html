消息推送服务moepush部署教程

发表于 2026-01-16 更新于 2026-01-22 分类于 软件 Waline： 阅读次数：

摘要: 消息推送服务moepush部署教程

项目介绍

  1. 项目地址https://github.com/beilunyang/moepush
  2. 项目介绍:一个基于 NextJS + Cloudflare 技术栈构建的可爱消息推送服务, 支持多种消息推送渠道
  3. 视频教程https://www.bilibili.com/video/BV1dtZBYnEUX/?p=2

项目部署

fork项目到自己仓库

创建个OAuth App

点击github右上角头像-settings-最下边Developer settings-OAuth Apps-new OAuth App

Application name随便起

Homepage URL填写部署后的实际域名比如https://123.com(填错了后续可更改)

Authorization callback URL填写(域名根据实际的替换)https://123.com/api/auth/callback/github

之后点击Register application

复制和Client secrets(新生成一个即可)备用

获取cloudflare信息

在cloudflare主页右上角头像点-配置文件-api令牌-创建令牌

基于编辑 Cloudflare Workers模版创建 额外添加D1数据库编辑权限

创建后备份api令牌

在cloudflare主页 邮箱的末尾的三个点-复制账户ID 备份

配置仓库变量

点击fork后的仓库-Settings-Secrets and variables-actions

在Repository secrets的右侧点击New repository secret新增如下变量

CLOUDFLARE_API_TOKEN上边创建的API 令牌Cloudflare API 令牌
CLOUDFLARE_ACCOUNT_ID上边获取的账户IDCloudflare Account ID
AUTH_SECRET加密 Session 的密钥随便给一个UUID
AUTH_GITHUB_ID上边的Client IDGitHub OAuth App ID
AUTH_GITHUB_SECRET上边的Client secretsGitHub OAuth App Secret
D1_DATABASE_NAMEmoepush-dbD1 数据库名称,随便起名
PROJECT_NAMEmoepush选填,在cloudflare中显示的项目名
DISABLE_REGISTERfalse选填,选填默认开启注册

部署

手动触发工作流。仓库 前往 Action页面，点击 Deploy 工作流，点击 Run workflow 按钮即可

等待部署成功 在cloudflare pages里选中已部署的项目 绑定域名即可(GitHub OAuth App中配置的相同的域名)
