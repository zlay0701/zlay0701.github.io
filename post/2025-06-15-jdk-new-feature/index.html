摘要: jdk选择和新特性记录

简介

所有特性:http://openjdk.java.net/jeps/0

LTS版本：Oracle官方长期支持版本（如JDK 8、11、17、21），企业环境中更常见。

发布周期：自JDK 9起，每6个月发布一个版本，每2年推出一个LTS版本。

2025年9月已发布JDK25 LTS版本

jdk选择

  1. 一般选LTS版本的jdk
  2. 一般选Eclipse Adoptium, 标准的openjdk实现,可以商用,也适合个人使用
  3. 需要openj9就选ibm的jdk
  4. 因为从jdk9开始javafx就不在jdk中了 需要带额外javafx插件的就选Azul Zulu或者Liberica
  5. 有的厂商也会单独编译jre,比如Adoptium, ibm, Zulu等等,如果需要jre就选这种版本
  6. jdk6和7只能去Oracle官网下载
  7. Oraclejdk8 最后可商用的版本是8u201

  1. Oraclejdk8 最后可商用的版本是8u201,8u201之后的新版本不可以商用,建议用openjdk
  2. Oraclejdk6(6u45)和7(7u80)已不在更新,不建议使用
  3. Oracle11禁止商用
  4. 从Oraclejdk17开始才限时商用的,Oraclejdk9-16都不可以商用
  5. Oraclejdk17.0.12是最后的可商用jdk17版本,之后的版本不可以商用
  6. Oraclejdk21有限商用到2026年9月,之后的版本禁止商用
  7. Oraclejdk25有限商用到2028.9月,之后的版本禁止商用
  8. Oracle的其他非lts版本jdk(jdk17之后的版本) 在新的jdk版本发布出来之后就不维护了,基本维护期就6个月

GraalVM

GraalVM是一个通用的虚拟机，用于运行以JavaScript、Pytho n、Ruby、R、JVM等语言编写的应用程序，如Java、Scala、Groovy、Kotlin 、Crojule和基于LLVM的语言，如C和C++。GraalVM消除了编程语言之间的隔离，并在共 享运行时启用互操作性。它可以独立运行，也可 以在OpenJDK、Node.js或Oracle数据库的上下文中运行。

openjdk 可商用版本有哪些(豆包AI)

OpenJDK 本身是基于 GPLv2+CE 许可协议开源的，可免费用于商业用途。此外，还有许多基于 OpenJDK 构建的可商用版本，以下是一些常见的介绍：

  1. Eclipse Temurin：由 Eclipse 开源基金会旗下的 Adoptium 工作组维护，之前是 AdoptOpenJDK。它是基于 OpenJDK 构建的免费的 JDK 发行版，可用于在 Java 生产环境中替代 Oracle JDK，提供长期支持，支持 LTS 和非 LTS 版本。
  2. Amazon Corretto：是由亚马逊维护的基于 OpenJDK 的免费的 JDK 生产就绪版本，提供长期支持，其中包括性能增强和安全修复。
  3. Azul Zulu：Azul 为许多操作系统和架构提供名为 Azul Zulu 的开源 OpenJDK 构建，此外，Azul 还提供满足特殊要求的构建，例如精简的 JRE 和包含 OpenJFX 的构建。Azul Zulu 有免费版本和收费的商业支持版本。
  4. BellSoft Liberica JDK：BellSoft 为几乎所有操作系统和架构提供名为 Liberica JDK 的开源 OpenJDK 构建，Liberica JDK 提供两种版本，开源免费版基于 GPLv2+CE，允许在生产环境中使用，商业支持版则需付费。
  5. Microsoft OpenJDK：2021 年，微软发布了 OpenJDK 的 Microsoft Build，这是另一个 OpenJDK 构建，为主要开发平台提供版本，使用 MIT 协议，可免费商用。
  6. Tencent Kona：Kona 是腾讯开源的、免费的、基于 OpenJDK 的发行版本，并且受腾讯长期支持（LTS）和季度更新，目前由腾讯专业的 JVM 技术团队进行维护和开发，主要应用于腾讯云业务场景及其他 Java 应用场景。
  7. Alibaba Dragonwell：是由阿里巴巴主导开源的 OpenJDK 发行版，阿里巴巴的主要业务都运行在基于 Linux 和 Dragonwell 的 Java 应用程序上，可免费用于商业用途。
  8. Huawei OpenJDK（毕昇 JDK）：是华为内部基于 OpenJDK 定制的 Huawei JDK 的开源版本，运行在华为内部 500 多个产品上，是一款高性能、可用于生产环境的 OpenJDK 发行版，在 ARM 架构上进行了性能优化和稳定性增强。

如何选择适合自己的OpenJDK商用版本？(豆包AI)

选择适合自己的 OpenJDK 商用版本需要结合实际需求综合考量，以下是关键的决策因素和建议：

支持周期与更新维护**

  • 长期支持（LTS）需求

    ：企业级应用通常优先选择 LTS 版本（如 JDK 8、11、17、21），需确认厂商提供的支持周期是否覆盖应用生命周期。例如：

    • Eclipse Temurin、Amazon Corretto 提供 8 年以上 LTS 支持；
    • Azul Zulu、BellSoft Liberica 的付费版本可提供更长定制支持。

  • 更新频率：需确保厂商能及时推送安全补丁（如 Log4j 类漏洞修复）和性能优化，避免依赖社区版的非定期更新。

平台兼容性**

  • 根据部署环境选择：
    • 服务器架构：ARM 架构（如 AWS Graviton、华为鲲鹏）优先考虑 Alibaba Dragonwell、Huawei 毕昇 JDK（针对性优化）；
    • 容器 / 云环境：BellSoft Liberica Lite（体积小）、Amazon Corretto（云原生适配）更合适；
    • 嵌入式系统：Liberica JDK Full（含 JavaFX）或 Azul Zulu Embedded 版本更优。

功能需求**

  • 特殊功能
    • 需 JavaFX：优先 Liberica JDK Full、Azul ZuluFX；
    • 性能优化：Alibaba Dragonwell（电商场景优化）、Tencent Kona（云原生增强）；
    • 轻量部署：选择精简版（如 Liberica Lite、Azul Zulu Core）。

商业支持与服务**

  • 免费 vs 付费
    • 中小项目 / 非核心系统：可选用免费社区版（如 Temurin、Corretto），依赖社区文档和论坛支持；
    • 核心业务 / 金融级应用：建议购买商业支持（如 Azul Zulu Enterprise、Liberica Commercial），获取厂商直接技术支持、SLA 保障和定制化服务。

生态与兼容性**

  • 确认版本与现有框架 / 工具兼容（如 Spring、Maven），可优先选择市场占有率高的版本（如 Temurin、Corretto），减少兼容性问题。
  • 检查是否通过 Java SE 兼容性测试套件（TCK），确保符合 Java 标准。

企业背景与信任度

  • 优先选择技术积累深厚的厂商：
    • 云厂商用户：AWS 用 Corretto，阿里云用 Dragonwell，生态整合更顺畅；
    • 跨平台需求：BellSoft Liberica（支持最广架构）、Azul Zulu（多系统适配）更可靠。

总结建议

  • 初创企业 / 轻量应用：Eclipse Temurin（免费、社区活跃）或 Amazon Corretto（云友好）；
  • 云原生 / 容器部署：BellSoft Liberica Lite（体积优化）或 Tencent Kona（腾讯云生态）；
  • ARM 架构 / 国产化需求：Huawei 毕昇 JDK、Alibaba Dragonwell；
  • 核心业务 / 付费支持：Azul Zulu Enterprise（全面服务）或 Liberica Commercial（多平台保障）。

可先通过小范围测试验证性能和兼容性，再大规模部署。
