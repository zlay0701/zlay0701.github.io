CloudPaste部署

发表于 2025-08-08 更新于 2026-01-22 分类于 软件 Waline： 阅读次数：

摘要: CloudPaste部署

官网https://github.com/ling-drag0n/CloudPaste

教程https://doc.cloudpaste.qzz.io/guide/deploy-github-actions

我简单写写大概步骤和会踩哪些坑 详细的看官网文档

提前准备

步骤

Cloudflare 创建令牌

点击https://dash.cloudflare.com/profile/api-tokens

创建新的 API 令牌 选择”编辑 Cloudflare Workers“模板，并添加 D1 数据库编辑权限

Cloudflare 账户id

点击左上角图标回到主页

点击主页邮箱尾部的三个点 复制账户id

fork 仓库配置环境变量

在您 Fork 的仓库中，进入 Settings → Secrets and variables → Actions->new Repository secrets，添加以下 Secrets：

Secret 名称必需说明
CLOUDFLARE_API_TOKEN✅Cloudflare API 令牌
CLOUDFLARE_ACCOUNT_ID✅Cloudflare 账户 ID
ENCRYPTION_SECRET❌加密密钥（可选，系统会自动生成）

部署后端到Cloudflare works

进入 Actions 标签页
选择 “Deploy Backend to Cloudflare Workers “ 工作流
点击 “Run workflow”
等待部署完成
工作流会自动：

创建 D1 数据库
初始化数据库结构
部署 Worker 到 Cloudflare
设置环境变量

配置后端域名(建议)

选中后端-Setting-自定义域

部署前端到Cloudflare pages

在github Action里运行 部署前端工作流Deploy Frontend to Cloudflare Pages

等待工作流运行完成

配置前端环境变量

在cloudFlare pages 选中cloudpaste-frontend-设置-变量和机密

添加环境变量：

  • 名称: VITE_BACKEND_URL
  • 值: 您的后端 Worker URL（如 https://cloudpaste-backend.your-username.workers.dev）

注意 环境变量值必须是完整的 URL，包含 https:// 前缀，末尾不要添加 /!!!!

之后重新部署前端(建议)

给前端配置域名

验证部署

  • 验证后端 访问 https://your-backend-url.workers.dev/api/health 域名缓存你自己的后端域名
  • 访问前端域名,会看到部署好的页面

配置R2存储

  • 新建个R2存储桶

  • 创建R2 api令牌

  • 复制 访问密钥ID 和 机密访问密钥 和 端点地址

  • 给R2 配置跨域编辑 CORS 策略,如下

在CloudPaste配置R2

https://doc.cloudpaste.qzz.io/guide/s3-config#%E5%9C%A8-cloudpaste-%E4%B8%AD%E9%85%8D%E7%BD%AE

  • 访问<前端域名>/admin
  • 默认管理员账号密码 admin/admin123

测试功能

  • 使用默认管理员账号登录
  • 创建测试文本
  • 上传测试文件(依赖上边的R2存储桶)
